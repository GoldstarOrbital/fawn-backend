API DOCUMENTATION
================================================================================

POST /wallet/create
  Authentication: Required (Bearer token)
  Rate Limit: 5/minute per user
  Status Codes:
    201 Created - Wallet created successfully
    400 Bad Request - User already has wallet, invalid wallet_type
    401 Unauthorized - Not authenticated
    422 Unprocessable Entity - Invalid request schema
    500 Internal Server Error - Derivation/encryption failed

  Request Schema:
    {
      "wallet_type": "non_custodial" | "fawn_custodial" (required)
    }

  Response Schema (CreateWalletResponse):
    {
      "wallet_address": "0x..." (string, EIP-55 checksummed),
      "wallet_type": "non_custodial" | "fawn_custodial" (string),
      "usdc_balance": 0.0 (float, in USD),
      "chain": "polygon" | "ethereum" (string),
      "seed_phrase": "word1 word2 ... word12" | null (string or null)
    }

  Seed Phrase Rules:
    - Non-custodial: Always returned (user must save)
    - Custodial: Always null (FAWN holds encrypted key)
    - Returned only once (not retrievable later)
    - 12 English words, space-separated

GET /auth/me
  Returns user profile including wallet persistence:
    {
      ...,
      "wallet_initialized": true,
      "crypto_wallet_address": "0x...",
      "wallet_type": "non_custodial" | "fawn_custodial",
      ...
    }

USAGE EXAMPLES
================================================================================

1. CREATE NON-CUSTODIAL WALLET (User-Managed)

   curl -X POST http://localhost:8000/wallet/create \
     -H "Authorization: Bearer $TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"wallet_type": "non_custodial"}'

   Response:
   {
     "wallet_address": "0x742d35Cc6634C0532925a3b844Bc2e7a1b8Bef6A",
     "wallet_type": "non_custodial",
     "usdc_balance": 0.0,
     "chain": "polygon",
     "seed_phrase": "abandon ability able about above absent absorb abstract abuse access accident account"
   }

   Frontend MUST:
   - Show seed phrase prominently (alert box, modal)
   - Require user to screenshot or download backup
   - Prevent copy/paste (or allow with warning)
   - Clear seed from memory after 30 seconds of inactivity
   - Store in localStorage temporarily for recovery

2. CREATE CUSTODIAL WALLET (FAWN-Held)

   curl -X POST http://localhost:8000/wallet/create \
     -H "Authorization: Bearer $TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"wallet_type": "fawn_custodial"}'

   Response:
   {
     "wallet_address": "0x6b175474e89094c44da98b954eedeac495271d0f",
     "wallet_type": "fawn_custodial",
     "usdc_balance": 0.0,
     "chain": "polygon",
     "seed_phrase": null
   }

   Frontend MUST:
   - Prompt user to set PIN (4 digits, future: biometric)
   - Store PIN hash locally (never send PIN to backend)
   - Explain that FAWN holds the key

3. CHECK WALLET STATUS

   curl -H "Authorization: Bearer $TOKEN" \
     http://localhost:8000/auth/me | jq '.wallet_initialized'

   Returns: true (wallet exists) or false (not yet created)

================================================================================
SUPPORT & TROUBLESHOOTING
================================================================================

ISSUE: "Encryption failed: FAWN_ENCRYPTION_KEY environment variable not set"

  Cause: Environment variable not configured in Railway
  Solution:
    1. Generate key: python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
    2. Add to Railway Variables: FAWN_ENCRYPTION_KEY=<key>
    3. Restart app: railway deployment list → select latest → redeploy

ISSUE: "Failed to derive wallet from seed phrase: ..."

  Cause: eth-account library not installed or seed format invalid
  Solution:
    1. Verify requirements.txt has eth-account>=0.10.0
    2. Check railway logs: railway logs -f | grep "Failed to derive"
    3. Re-push if needed: git push origin main

ISSUE: Seed phrase is not 12 words

  Cause: Mnemonic library not generating correctly
  Solution:
    1. Test locally: python3 -c "from mnemonic import Mnemonic; m = Mnemonic('english'); print(m.generate(strength=128))"
    2. Should output exactly 12 words
    3. If not, check mnemonic version: pip show mnemonic

ISSUE: Custodial wallet returns seed_phrase instead of null

  Cause: create_wallet() not returning None for custodial
  Solution:
    1. Check services/crypto_wallet.py line: "seed_phrase": seed_phrase if wallet_type == "non_custodial" else None
    2. Verify condition is correct
    3. Add logging to confirm wallet_type value
