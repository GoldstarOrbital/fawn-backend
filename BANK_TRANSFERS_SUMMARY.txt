WHAT WAS BUILT
================================================================================

New API Endpoint:
  POST /transfers/send-to-bank

  Allows authenticated FAWN users to send USDC to ANY bank account via ACH.
  Same $0.01 flat fee as P2P transfers. Settlement: 1-3 business days.

Capabilities Enabled:
  ✓ Send $X USDC from FAWN wallet
  ✓ Convert USDC → USD (1:1, no slippage)
  ✓ Initiate ACH transfer to recipient's bank account
  ✓ Deduct balance immediately (pessimistic)
  ✓ Track transfer with pending status
  ✓ Audit log with 7-year compliance retention

Security:
  ✓ Input validation (routing number format, account length, etc.)
  ✓ Balance checks (amount + $0.01 fee must be available)
  ✓ Bank details NOT persisted (only last 4 digits stored)
  ✓ Idempotency keys prevent retry duplicates
  ✓ Audit trail with truncated recipient info
  ✓ Rate limiting per user

KEY FEATURES
================================================================================

1. Pessimistic Balance Model
   - Balance decreases immediately when transfer initiated
   - User sees immediate change (won't overdraft)
   - If ACH fails later, balance is refunded

2. Security
   - Input validation: routing (9 digits), account (4-17 digits)
   - Full bank details NOT stored (only last 4 of account)
   - Idempotency keys prevent duplicate ACH entries
   - Audit trail with 7-year compliance retention

3. Error Handling
   - Insufficient balance (402)
   - No wallet (404)
   - Column unavailable (503 with refund)
   - ACH API errors (503 with refund)
   - Input validation (422)

4. Fee Model
   - Flat $0.01 per transfer (same as P2P)
   - No hidden ACH fees
   - No gas fees (not blockchain)

5. Settlement Time
   - Status: pending (1-3 business days)
   - Column handles ACH routing/clearing
   - Webhook handler will update status on completion (TODO)
