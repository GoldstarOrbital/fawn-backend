2. CORE SERVICE LOGIC CHANGES
================================================================================

File: services/crypto_wallet.py

CHANGES MADE:

A) NEW FUNCTION: _derive_wallet_from_seed(seed_phrase: str) -> tuple[str, str]

   PURPOSE: Derive Ethereum wallet address + private key from BIP39 seed phrase

   IMPLEMENTATION:
     from eth_account import Account
     from eth_utils import to_checksum_address

     account = Account.from_mnemonic(seed_phrase, account_path="m/44'/60'/0'/0/0")
     address = to_checksum_address(account.address)  # EIP-55 checksummed
     private_key = account.key.hex()  # Returns "0x..." format
     return address, private_key

   OUTPUT EXAMPLE:
     address: "0x742d35Cc6634C0532925a3b844Bc2e7a1b8Bef6A"
     private_key: "0x1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef"

B) UPDATED FUNCTION: _encrypt_private_key(private_key: str) -> bytes

   BEFORE: Placeholder error handling
   AFTER: Real Fernet encryption

   KEY POINTS:
     - Uses FAWN_ENCRYPTION_KEY environment variable
     - Fernet = AES-256-GCM with automatic IV + HMAC
     - Returns bytes (stored as BYTEA in database)
     - Includes try/except with meaningful error messages

C) UPDATED FUNCTION: create_wallet(user_id, db, wallet_type) -> dict

   KEY CHANGES:

     1. REAL SEED GENERATION:
        seed_phrase = _generate_seed_phrase()  # 12 words via mnemonic

     2. REAL WALLET DERIVATION:
        wallet_address, private_key = _derive_wallet_from_seed(seed_phrase)
        # Now returns EIP-55 checksummed address + hex private key

     3. ENCRYPTED KEY STORAGE:
        if wallet_type == "fawn_custodial":
            encrypted_key = _encrypt_private_key(private_key)
        # Uncommented and now properly stores encrypted keys

     4. DATABASE INSERT:
        wallet = CryptoWallet(
            user_id=user_id,
            wallet_address=wallet_address,      # Real derived address
            wallet_type=wallet_type,
            chain=USDC_CHAIN,
            usdc_balance_cents=0,
            encrypted_private_key=encrypted_key,  # Now properly set
        )
        db.add(wallet)

     5. AUDIT LOGGING:
        # Logs wallet_address in audit trail (but NOT seed or private key)
        details=json.dumps({
            "wallet_type": wallet_type,
            "chain": USDC_CHAIN,
            "wallet_address": wallet_address,  # Safe to log (public)
        })

     6. RETURN VALUE:
        {
            "wallet_address": "0x...",
            "wallet_type": "non_custodial"|"fawn_custodial",
            "usdc_balance": 0.0,
            "chain": "polygon",
            "seed_phrase": "word1 word2 ... word12" if non_custodial else None
        }

SECURITY IMPLICATIONS:
  - Private keys encrypted with AES-256-GCM before database storage
  - Seed phrases only returned for non_custodial (never logged)
  - Both never exposed in audit logs or error messages
  - 7-year audit trail for compliance
